    <div>
      <h2 class="text-2xl font-display font-extrabold text-white mb-3">Project Overview</h2>
      <p>
        The EOD (Explosive Ordnance Disposal) Trainer is an interactive simulation system built to demonstrate the capabilities of distributed wireless hardware nodes. A mobile rover equipped with a 3D-printed 4-DOF robotic arm is operated wirelessly. The goal: navigate to a target prop, identify the correct wire defusal sequence, and safely extract the wires before the countdown expires.
      </p>
    </div>

    <div>
      <h2 class="text-2xl font-display font-extrabold text-white mb-3">System Architecture & Node Diagram</h2>
      <p class="mb-4">
        To maintain low-latency responsiveness without relying on a central Wi-Fi access point, the system operates completely peer-to-peer using Espressif's <strong>ESP-NOW protocol</strong>. Each ESP32 microchip operates as a specific functional node:
      </p>
      
      {/* Dynamic Diagram Mounting */}
      <RobotDiagram client:load />

      <ul class="list-disc pl-6 space-y-2 mt-4">
        <li><strong>Controller Node (The Remote):</strong> Reads analog joysticks and tactile buttons, manages the active game states (Timer, Lives, Defusal sequences), and prints outputs to a 1602 LCD display.</li>
        <li><strong>Rover Node (The Robot):</strong> Directs four 12V DC metal gear motors via L298N drivers for holonomic Mecanum wheel driving, actuates robotic arm servos, and reads the accelerometer.</li>
        <li><strong>Target Node (The Bomb Prop):</strong> Monitors the physical wire switches and wirelessly transmits state updates to the controller.</li>
      </ul>
    </div>

    <div>
      <h2 class="text-2xl font-display font-extrabold text-white mb-3">Key Technical Challenges</h2>
      
      <div class="space-y-4 mt-4">
        <div class="bg-slate-950/40 border border-slate-900 rounded-xl p-5">
          <h4 class="text-sm font-bold text-white mb-1">1. Non-Blocking ESP-NOW Tasking</h4>
          <p class="text-slate-400 text-xs">
            Using connectionless packet streams requires robust frame parsing to avoid blocking the FreeRTOS loop. Packets are parsed asynchronously using structs on both ends of the transmission, guaranteeing that motor PWM signals update in real-time under 20ms of jitter.
          </p>
        </div>

        <div class="bg-slate-950/40 border border-slate-900 rounded-xl p-5">
          <h4 class="text-sm font-bold text-white mb-1">2. Tremor Penalty Algorithm (MPU6050)</h4>
          <p class="text-slate-400 text-xs">
            To simulate high-stakes stress, an MPU6050 accelerometer is mounted directly on the robotic claw. The rover continuously samples 3-axis acceleration. A moving standard deviation exceeds the configured threshold when shaking occurs, triggering immediate game time-penalties.
          </p>
        </div>
      </div>
    </div>

    <div>
      <h2 class="text-2xl font-display font-extrabold text-white mb-3">Key Takeaways</h2>
      <p>
        This project highlights the strength of building distributed embedded logic. Abandoning bulky serial cables in favor of localized radio packets allowed clean, real-time control pathways, showing that microcontrollers can orchestrate complex, low-cost robotics operations.
      </p>
