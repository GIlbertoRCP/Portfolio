      <h2 class="text-2xl font-display font-extrabold text-white mb-3">System Concept</h2>
      <p>
        Traditional Software-Defined Networks (SDN) monitor traffic and apply routing changes <em>reactively</em> once congestion has already occurred. This project introduces a decoupled, streaming-first architecture that predicts queue spikes ahead of time, scaling up switch polling rate and installing proactive OpenFlow rules to steer traffic away from congestion points.
      </p>

    <div>
      <h2 class="text-2xl font-display font-extrabold text-white mb-3">Core Architecture</h2>
      <p>
        The system decouples data telemetry, inference, and switch rule modification into independent microservices running on a Python 3.12 stack:
      </p>
      
      <ul class="list-disc pl-6 space-y-2 mt-2">
        <li><strong>SDN Control Plane:</strong> Built using <em>os-ken</em> (compatible Ryu fork) running alongside Open vSwitch in Mininet. It publishes port stats to a Kafka topic and maintains an under 30MB control footprint.</li>
        <li><strong>Kafka Event Bus:</strong> KRaft-mode broker handling standard messages on <code>network-telemetry</code> and <code>network-control</code> topics.</li>
        <li><strong>Inference Service:</strong> FastAPI microservice managing PyTorch models. An LSTM network evaluates sliding windows of port metrics to predict 5-step-ahead queues, and a DQN agent selects polling rates based on predicted risk.</li>
      </ul>
    </div>
