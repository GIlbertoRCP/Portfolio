    {/* Section 1: The Problem */}
    <div class="space-y-4">
      <h2 class="text-2xl font-display font-extrabold text-white flex items-center gap-2">
        <span class="text-blue-500 text-xl font-mono">01/</span> The Network Monitoring Problem
      </h2>
      <p>
        Traditional Software-Defined Networks (SDN) monitor traffic and apply routing changes <strong>reactively</strong>. Once a switch queue is saturated and latency spikes, packet loss has already occurred. Polling switches continuously for statistics solves this but creates massive CPU and network overhead on the controller plane.
      </p>
      <p>
        To balance this trade-off, this system decouples telemetry collection, inference calculations, and flow rule modifications using an event-driven Kafka streaming pipeline. This keeps the controller lightweight (under 30MB) while maintaining millisecond-level responsiveness.
      </p>
    </div>

    {/* Section 2: Decoupled Kafka Architecture */}
    <div class="space-y-4">

        <span class="text-blue-500 text-xl font-mono">02/</span> Decoupled Event-Driven Pipeline
      </h2>
      <p>
        The architecture splits network operations into four independent modules running on a Python 3.12 stack:
      </p>
      
      <div class="grid sm:grid-cols-3 gap-6 pt-4">

        <span class="text-blue-500 text-xl font-mono">04/</span> Machine Learning &amp; Edge Optimization

      <p>
        The self-healing logic relies on two distinct machine learning models:
      </p>
      <ul class="space-y-3.5 mt-4">
        <li class="flex items-start gap-3">
          <span class="text-blue-500 mt-1">✓</span>
          <div>
            <strong class="text-white">PyTorch LSTM Congestion Forecaster:</strong> 
            Processes sliding windows of port metrics to predict 5-step-ahead queue depth, catching spikes before they overflow switch buffers.
          </div>
        </li>
        <li class="flex items-start gap-3">
          <span class="text-blue-500 mt-1">✓</span>
          <div>
            <strong class="text-white">DQN Reinforcement Learning Agent:</strong> 
            Balances switch polling frequency (coarse 5s standby vs fine 1s intensive mode) against controller CPU overhead based on predicted congestion risks.
          </div>
        </li>
      </ul>
