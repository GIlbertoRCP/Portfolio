      <h2 class="text-2xl font-display font-extrabold text-white mb-3">System Engineering & Sync</h2>
      <p>
        Multiplayer rooms require strong transactional guarantees. We built the sync layer with:
      </p>
      
      <ul class="list-disc pl-6 space-y-2 mt-2">
        <li><strong>Dynamic Room Routing:</strong> Leverages Next.js parallel layouts (<code>(public)</code> and <code>(host)</code> namespaces) to split host dashboards from general player join views.</li>
        <li><strong>State Polling Loop:</strong> Implements a client-side polling ticker checking <code>/api/rooms/[roomId]/state</code>, syncing question changes and timer bars under 100ms lag thresholds.</li>
        <li><strong>Millisecond Timing verification:</strong> When a user clicks or presses a key, the client captures the precise timestamp relative to the round start, sending the payload to the database to prevent network latency cheating.</li>
      </ul>
    </div>
